# 00-02. 문서 체계 및 작업 계획

## 개요

본 프로젝트의 문서는 모듈별로 단계적인 개발을 진행하기 위해 체계적인 번호링 시스템을 따른다. 기획부터 디자인, 개발 계획까지 일관된 흐름으로 관리한다.

---

## 문서 번호 체계

**형식**: `XX-YY_이름`

### XX: 모듈/카테고리

- **00**: 프로젝트 전반 (소개, 가이드 등)
- **01**: 콘텐츠 매니저
- **02**: Review
- **03**: Note
- **04**: Playground
- **05**: 점수 시스템
- **06**: 칭호 시스템
- **07**: 소셜 시스템
- **08**: 콘텐츠 피드
- **09**: 탐험

### YY: 문서 유형

#### 01: 기획서 (기능 명세)
- 해당 모듈의 기능, 요구사항, 데이터 모델 등을 정의한 문서
- 예: `01-01_콘텐츠 매니저.md`

#### 02: 컨셉 아트 (HTML/CSS 프로토타입) 
- 기획 내용을 바탕으로 시각적 디자인과 인터랙션을 구현한 HTML 파일
- 실제 개발 전 UI/UX를 검증하고 디자인 방향성을 잡는 단계
- 하나의 모듈에 여러 화면이 필요한 경우, 파일명 뒤에 상세 구분을 붙여 확장
- 예: 
  - `01-02_기록관_그리드뷰.html`
  - `01-02_기록관_리스트뷰.html`
  - `01-02_콘텐츠_상세.html`

#### 03: 개발 기획서 (구현 설계)
- 컨셉 아트와 기획서를 바탕으로 실제 구현을 위한 기술적 계획 수립
- 컴포넌트 구조, 상태 관리, API 명세, 데이터베이스 스키마 상세 등 포함
- 예: `01-03_콘텐츠 매니저_개발계획.md`

---

## 작업 프로세스

1. **기획 단계 (01)**
   - 각 모듈별 기능 정의 및 스펙 확정 (현재 완료 단계)

2. **컨셉 아트 단계 (02)**
   - 기획서를 바탕으로 HTML/CSS 컨셉 아트 제작
   - 디자인 퀄리티와 사용자 경험(UX) 최우선 고려
   - **"Rich Aesthetics"**, **"Dynamic Design"** 원칙 적용

3. **개발 기획 단계 (03)**
   - 확정된 디자인과 기능을 토대로 구체적인 구현 로드맵 작성

4. **구현 단계**
   - Next.js/React 기반 실제 애플리케이션 개발

---

## 향후 계획

현재 작성된 `01` 계열의 기획서들을 바탕으로, 각 모듈의 `02` 컨셉 아트를 순차적으로 제작해 나간다.
