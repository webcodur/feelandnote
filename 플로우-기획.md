# 플로우(Flow) 기능 최소 viable 스펙  
(직선형 기반, 초보 큐레이터도 쉽게 멋진 결과물 만들기 중심)

## 왜 플로우 기능이 필요한가? (배경과 의도)

### 현재 서비스의 현실적인 문제
- 지금은 재생목록(플레이리스트)만 존재  
  → 컨텐츠를 모아놓기만 할 뿐, **순서에 대한 어떠한 가이드나 의도도 없음**
- 사용자가 느끼는 가장 큰 불편:  
  **"이걸 왜 이 순서로 봐야 하지?"**  
  → 답이 전혀 없음. 순서가 임의로 바뀌거나 무시되어도 문제없는 단순 리스트

### 플로우로 바꾸고 싶은 목표
- 단순 재생목록 → **"이 순서대로 따라가면 이런 경험을 할 수 있다"는 약속**으로 업그레이드
- 사용자 가치: 시간 절약 + 맥락 있는 몰입 + 완주 성취감
- 생성자 가치: 자신의 의도·취향·스토리를 제대로 전달할 수 있음
- 비즈니스 기대: 체류 시간·재방문·리뷰 증가 + 니치 콘텐츠 재발견

## 기본 구조
- 플로우는 **직선형(Sequential)**으로만 구성  
  → 시작 → 노드1 → 노드2 → … → 끝  
  분기·트리·그래프·사이클·조건 분기 없음
- 하나의 플로우 = 컨텐츠 노드들의 순차적 리스트  
  (책, 영화, 음악 앨범/트랙 자유 혼합 가능)

## 핵심 기능 명세 (최소한으로만)

### 1. 플로우 기본 정보
- 제목 (필수)
- 한 줄 설명 (선택)
- 전체 테마 색상 팔레트 (필수)  
  → 전체 UI 톤 반영
- 예상 소요 시간 (선택)
- 난이도 (1~5 ★, 선택)

### 2. 노드(Node) 구성 요소
- 컨텐츠 선택 (필수)  
  → 기존 책/영화/음악 중 하나
- 노드별 설명 (선택)  
  → 왜 이 순서에 이 작품인지, 이전과의 연결 등 자유 작성
- 보너스 콘텐츠 (선택)  
  → 추가 컨텐츠 링크 (같은 타입 or 다른 타입)
- 노드별 난이도 (선택, 1~5 ★)
- 노드별 테마 색상 오버라이드 (선택)

### 3. 플로우 전체 꾸미기 요소 
- 전체 커버 이미지 (자동 생성 or 수동)
- 완주 인증 문구/뱃지 (선택, 한 줄 커스텀)

## 기획 의도 요약
- **"이걸 왜 이 순서로 봐야 하지?"** 질문에 답하는 도구
- 직선형 단순함 유지하면서도 생성자가 스토리·의도를 담을 수 있게
- 결과물이 시각적으로 세련되고 내용적으로 알차 보이게
- 사용자에게 "이 플로우 하나만 따라가면 제대로 즐길 수 있겠다"는 신뢰감 제공
