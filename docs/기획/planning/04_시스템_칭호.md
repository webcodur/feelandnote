# 06-01. 칭호 시스템

## 개요

칭호 시스템은 특정 조건 달성 시 해금되는 숨겨진 업적이다. 조건은 비공개로 예상치 못한 순간에 서프라이즈로 등장하며, 해금 시 보너스 점수가 부여된다.

---

## 핵심 원칙

### 조건 비공개
- 해금 전까지 조건 노출 안 함
- 업적서에서 미해금 칭호는 "???" 표시

### 서프라이즈 요소
- 예상치 못한 순간에 팝업으로 해금 알림
- 해금 시 칭호명 + 설명 공개

### 영구 보존
- 한 번 해금된 칭호는 영구 보존
- 삭제/취소 불가

## 해금 로직

### 체크 시점
- 관련 활동 발생 시 실시간 체크
- 예: 콘텐츠 추가 시 → 기록량 계열 칭호 조건 체크

### 조건 체크 예시

**"백 권의 무게" 해금 로직**
```
IF user.contentCount >= 100 AND NOT hasTitle("백 권의 무게")
  THEN unlockTitle("백 권의 무게")
```

**"올빼미족" 해금 로직**
```
IF user.nightRecordCount >= 50 AND NOT hasTitle("올빼미족")
  // nightRecordCount: 02:00-05:00 사이 기록 횟수
  THEN unlockTitle("올빼미족")
```

## 데이터 모델 (개념)

### Title (칭호 정의)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 고유 식별자 |
| name | string | 칭호명 |
| description | string | 해금 후 설명 |
| category | enum | 카테고리 |
| grade | enum | 등급 |
| bonusScore | number | 보너스 점수 |
| condition | object | 해금 조건 (내부용) |

### UserTitle (유저 보유 칭호)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 고유 식별자 |
| userId | string | 유저 ID |
| titleId | string | 칭호 ID |
| unlockedAt | datetime | 해금일시 |

## 연동 모듈

- **점수 시스템**: 칭호 해금 시 보너스 점수 적립
- **콘텐츠 매니저**: 콘텐츠 추가 시 기록량 칭호 체크
- **Review/Note/창작**: 작성 시 관련 칭호 체크
- **소셜 시스템**: 팔로워 변동 시 소셜 칭호 체크
